# Story 07A.2 — Redis em dev local (via Docker)

**Status:** Draft
**Phase:** 07A (Mastra core)
**Owner agent:** `@devops` (Operator)
**Story type:** Infrastructure
**Estimated effort:** 20-30 min

## Context

O design da Phase 07 escolhe **BullMQ + Redis** como pattern de processamento de mensagens (Decisão 4 aprovada pelo CEO: "opção C sem dúvidas, escala desde dia 1"). BullMQ depende de Redis como backend de filas.

**Escopo decidido (2026-04-19):** configuração **dev local apenas**. Redis em produção (Coolify) fica documentado como playbook mas é executado na fase de preparação pra deploy real (Phase 12/13 ou quando Vinni sinalizar). Motivo: nenhuma phase da 07 vai rodar em produção; configurar Coolify agora é trabalho que não será testado por semanas.

Docker Desktop já está aberto na máquina do Vinni — infra host confirmada.

## Acceptance Criteria

### Dev local (obrigatório agora)

1. **Redis 7+ rodando localmente** via Docker Compose em `docker-compose.dev.yml`
   - Imagem: `redis:7-alpine`
   - Porta: `6379` mapeada pra host (localhost:6379)
   - Volume persistente nomeado (ex: `vertech-redis-data`) pra não perder jobs em `docker-compose down`
   - AOF habilitado (`appendonly yes`) — persistência em disco
   - Auto-restart: `restart: unless-stopped`
2. **Variável `REDIS_URL`** configurada:
   - `.env.example`: linha `REDIS_URL=redis://localhost:6379` com comentário explicativo
   - `.env` local do Vinni: mesmo valor (não commitar)
3. **Teste funcional:** `docker compose -f docker-compose.dev.yml exec redis redis-cli ping` retorna `PONG`
4. **Teste de persistência:** subir Redis → `SET foo bar` → `docker compose down` → `docker compose up` → `GET foo` retorna `bar`
5. **Comando `pnpm redis:start`** (ou similar) adicionado ao `package.json` raiz como atalho pra subir só o Redis (útil pra @dev não rodar stack inteira)

### Playbook futuro (documentação, sem execução agora)

6. `docs/infrastructure/redis.md` criado com 2 seções claras:
   - **Seção 1: Dev local** (passo-a-passo do que foi feito acima)
   - **Seção 2: Coolify (futuro)** — playbook copy-paste com:
     - Como criar Redis service no Coolify dashboard (prints ou passo-a-passo textual)
     - Imagem `redis:7-alpine`, volumes, env vars, health check nativo do Coolify
     - Configurar `REDIS_URL=redis://redis-service-name:6379` nas variáveis de ambiente do app
     - Segurança: porta 6379 **NÃO** exposta publicamente, só rede interna Coolify
     - Backup strategy (snapshot AOF, volume backup)
     - Tempo estimado: ~20-30 min quando for executado
7. Seção 2 explicitamente marcada como **"Executar na Phase 12/13 ou quando começar preparação pra deploy"** — não é tarefa pra agora

### Sem execução no Coolify agora

8. **NÃO** configurar nada no Coolify dashboard nesta story
9. **NÃO** modificar env vars do Coolify nesta story
10. `@devops` deixa o playbook pronto e aguarda sinal do Vinni quando for hora de deploy real
