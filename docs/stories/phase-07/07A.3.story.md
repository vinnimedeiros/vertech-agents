# Story 07A.3 — Schema Drizzle 07A (agents + ajustes)

**Status:** Draft
**Phase:** 07A (Mastra core)
**Owner agent:** `@data-engineer` (Dozer)
**Story type:** Database schema + migration
**Estimated effort:** 45-60 min

## Context

A Phase 07 introduz o conceito de **Agent** como entidade de primeira classe — cada workspace pode ter múltiplos agentes configuráveis. Também introduz **versionamento** (rollback de mudanças de configuração) e **estados de processamento** visíveis (`QUEUED`, `PROCESSING`) pra suportar o fluxo BullMQ.

Phase 06 já deixou `conversation.assignedAgentId` como `text` solto (sem FK) e `conversation.isAIEnabled` como boolean — preparação exatamente pra este momento.

## Acceptance Criteria

### Nova tabela `agent`

1. Arquivo: `packages/database/drizzle/schema/agents.ts`
2. Enum `AgentStatus`: `DRAFT | ACTIVE | PAUSED | ARCHIVED` (default `DRAFT`)
3. Colunas obrigatórias:
   - `id`, `organizationId` (FK cascade → `organization.id`)
   - `name` (notNull), `role`, `avatarUrl`, `gender`, `description`
   - `model` (default `openai/gpt-4.1-mini` — **ajustar baseado no resultado da 07A.1**)
   - `temperature` (real, default 0.7), `maxSteps` (int, default 10)
   - `personality` (jsonb, default `{}`), `businessContext` (jsonb, default `{}`), `conversationStyle` (jsonb, default `{}`) — tipados via types em `packages/database/types/agent.ts`
   - `instructions` (text, nullable — override manual do builder)
   - `enabledTools` (text array, default `{}`)
   - `knowledgeDocIds` (text array, default `{}`)
   - `status` (enum, default `DRAFT`), `version` (int, default 1)
   - `whatsappInstanceId` (text, nullable)
   - `createdAt`, `updatedAt`, `publishedAt` (nullable)
4. Índices:
   - `agent_org_status_idx` em (`organizationId`, `status`)
   - `agent_whatsapp_instance_idx` em `whatsappInstanceId`
5. Relations declaradas via Drizzle: `organization`, `versions` (1-N com `agent_version`)

### Nova tabela `agent_version`

6. Snapshot imutável pra rollback/audit
7. Colunas: `id`, `agentId` (FK cascade), `version` (int), `snapshot` (jsonb), `createdByUserId` (FK nullable → `user.id`), `createdAt`
8. Unique index: (`agentId`, `version`)

### Ajustes em tabelas existentes

9. `conversation.assignedAgentId`: transformar em FK real → `agent.id` ON DELETE SET NULL (hoje é text solto). **Importante:** isso exige migration cuidadosa — dados existentes com `assignedAgentId` apontando pra IDs fictícios podem quebrar a constraint; solução: nullear registros sem correspondência antes de aplicar FK.
10. `messageStatusEnum`: adicionar valores `QUEUED` e `PROCESSING` entre `PENDING` e `SENT`.
    - **Atenção:** alteração de enum em Postgres exige `ALTER TYPE ... ADD VALUE`. Drizzle pode não gerar isso automaticamente — revisar migration SQL gerado.

### Types tipados

11. Arquivo novo: `packages/database/types/agent.ts` exportando:
    - `AgentPersonality`: `{ tone?, formality?, humor?, empathyLevel? }`
    - `AgentBusinessContext`: `{ industry?, products?, pricing?, policies?, inviolableRules? }`
    - `AgentConversationStyle`: `{ greeting?, qualificationQuestions?, objectionHandling?, handoffTriggers? }`
    - `AgentSnapshot`: cópia completa do agent usada em `agent_version.snapshot`
12. Schema usa esses tipos via `.$type<AgentPersonality>()` etc

### Migration + RLS

13. Migration SQL gerada via `pnpm db:generate` e revisada manualmente pra garantir:
    - ALTER TYPE pra enum additions
    - FK constraint com SET NULL handling correto
    - Zero DROP de colunas existentes (aditivo)
14. RLS policies via migration SQL separada ou no mesmo arquivo:
    - `agent` SELECT: user é membro da `organizationId`
    - `agent` INSERT/UPDATE/DELETE: role `admin` ou `owner`
    - `agent_version` SELECT: herdado de `agent` via join

### Verificação

15. `pnpm db:migrate` aplica migrations com sucesso (dev)
16. `\d agent` no psql mostra estrutura esperada
17. `\dT AgentStatus` mostra os 4 valores enum
18. `\dT MessageStatus` mostra os 7 valores (PENDING, QUEUED, PROCESSING, SENT, DELIVERED, READ, FAILED)

### Exports

19. `packages/database/drizzle/schema/index.ts` atualizado exportando `./agents`
