# Story 07A.5 — Mastra core (instance + agente dinâmico + instructions)

**Status:** Draft
**Phase:** 07A (Mastra core)
**Owner agent:** `@dev` (Neo)
**Story type:** Feature — AI infrastructure
**Estimated effort:** 90-120 min

## Context

Coração da Phase 07: instalar Mastra sobre o AI SDK existente, configurar storage Postgres pra memória persistida, e implementar o **agente comercial dinâmico** que lê configuração do banco via `requestContext` (UM agente multi-tenant, não N agentes por workspace).

Esta story entrega **o cérebro** — mas ainda não o conecta ao WhatsApp. Conexão end-to-end é 07A.6.

## Acceptance Criteria

### Instalação

1. Instalar deps no `packages/ai`:
   - `@mastra/core`, `@mastra/memory`, storage Mastra Postgres (nome exato confirmado em 07A.1)
   - Versões conforme 07A.1
2. Preservar deps existentes: `@ai-sdk/openai`, `@ai-sdk/react`, `ai` continuam funcionando — AI SDK puro ainda é usado pela feature `AiChat` do boilerplate

### Estrutura `packages/ai/src/mastra/`

3. Criar estrutura conforme design spec:
   ```
   packages/ai/src/mastra/
   ├── index.ts
   ├── instance.ts
   ├── storage.ts
   ├── agents/
   │   └── commercial.ts

   ├── instructions/

   ├── memory/
   │   └── config.ts
   └── runtime/
       └── invoker.ts  (stub — implementado em 07A.6)
   ```

### `storage.ts`

4. Exporta `mastraStorage` — instância do adapter Postgres apontando pra `DATABASE_URL`
5. Config inclui schema de tabelas próprias Mastra (`mastra_*`) pra não colidir com schema da aplicação

### `memory/config.ts`

6. Exporta `commercialAgentMemory` — `new Memory({ options: { lastMessages: 20, observationalMemory: true } })`
7. Parameters confirmados em 07A.1 (ajustar se API mudou)

### `tools/commercial.ts`, `tools/architect.ts`, `tools/orchestrator.ts`

8. Cada um exporta registry vazio tipado:
   ```typescript
   export const commercialTools: Record<string, Tool> = {};
   ```
9. Comentário no topo explicando quando vai ser populado (Phase 08 / 09 / 10)

### `instructions/builder.ts`

10. Exporta `buildInstructions(agent: AgentRecord): string`
11. Se `agent.instructions` (override manual) tiver valor, retorna ele diretamente
12. Caso contrário, renderiza template a partir de `agent.personality`, `agent.businessContext`, `agent.conversationStyle`:
    - Seção "Você é {name}, {role}"
    - Seção "Personalidade" (tom, formalidade, humor, empatia)
    - Seção "Sobre o negócio" (indústria, produtos, pricing, políticas, regras invioláveis)
    - Seção "Como você conversa" (saudação, qualificação, objeções, handoff)
    - Seção "Regras INVIOLÁVEIS" (hardcoded): nunca invente preços/prazos, seja conciso, não se identifique como IA a menos que perguntem
    - Seção "Ferramentas disponíveis" (placeholder em 07A — vazio até Phase 08)
13. Template reutilizável com escape seguro (sem injection via campo do agent)

### `agents/commercial.ts`

14. Exporta `commercialAgent` — `new Agent({ ... })` do Mastra com:
    - `id: 'commercial-agent'`, `name: 'Commercial Agent'`
    - `model`, `instructions`, `tools` como **funções** que recebem `requestContext` e buscam config do banco
    - `model: async ({ requestContext }) => { const a = await db.query.agent.findFirst({ where: eq(agent.id, requestContext.get('agentId')) }); return a.model; }`
    - `instructions: async ({ requestContext }) => buildInstructions(agentFromDb)`
    - `tools: async ({ requestContext }) => filterTools(commercialTools, agentFromDb.enabledTools)` — retorna `{}` em 07A (registry vazio)
    - `memory: commercialAgentMemory`
15. Helper `filterTools(registry, enabledKeys)` que retorna subset

### `instance.ts`

16. Exporta `mastra` — `new Mastra({ agents: { commercialAgent }, storage: mastraStorage, telemetry: { enabled: true, serviceName: 'vertech-agents' } })`
17. Singleton exportado do package

### `index.ts`

18. Re-exporta:
    ```typescript
    export { mastra } from './mastra/instance';
    export { commercialAgent } from './mastra/agents/commercial';
    export { buildInstructions } from './mastra/instructions/builder';
    // invoker stub ainda, implementado em 07A.6
    export { invokeAgentForMessage } from './mastra/runtime/invoker';
    ```

### Qualidade

19. `pnpm typecheck` passa
20. Unit test: boot da Mastra instance, instanciação do agente, resolução de `model/instructions/tools` com mock de `requestContext`
21. Mastra Studio (`mastra dev`) inicializa sem erro em `http://localhost:4111` (dev local)
22. `MASTRA_STUDIO_ENABLED` env var configurado — default `true` em dev, `false` em `.env.example` pra prod

## Notes

- **Storage migration:** `@mastra/pg` (ou equivalente) cria suas próprias tabelas no primeiro boot. Isso é **esperado** e não conflita com Drizzle migrations.
- **Não usar `Agent` do AI SDK diretamente** — sempre via Mastra. AI SDK é dep transitiva.
- **Não implementar invoker ainda** — stub que lança `throw new Error('Not implemented — see 07A.6')`
- Se storage Postgres oficial não existir, fallback: `@mastra/libsql` apontando pra `DATABASE_URL` (o protocolo libsql suporta Postgres).
