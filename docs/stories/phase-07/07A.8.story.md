# Story 07A.8 — Seed de agente + Quality Gate 07A

**Status:** Draft
**Phase:** 07A (Mastra core)
**Owner agent:** `@dev` (Neo) + `@qa` (Oracle)
**Story type:** Seed + Quality Gate
**Estimated effort:** 45-60 min

## Context

Com toda a infra construída em 07A.1-07A.7, esta story entrega:

1. **Seed SQL/script** que cria um agente comercial de teste pronto pra uso + vincula a instância WhatsApp existente — permite Vinni validar visualmente sem UI (ainda não existe, é 07B).
2. **Quality gate completo** rodado pelo @qa antes do gate humano do Vinni.

Ao fim desta story, **Vinni recebe convite pra rodar gate humano** e validar pessoalmente no celular.

## Acceptance Criteria

### Seed script

1. Criar `packages/database/seeds/07a-agent-commercial.ts`
2. Script idempotente (roda múltiplas vezes sem duplicar): check por `name` + `organizationId`
3. Cria agente com dados realistas de exemplo:
   - `name: 'Atendente Comercial Vertech'`
   - `role: 'Atendimento comercial'`
   - `model: 'openai/gpt-4.1-mini'` (ID exato da 07A.1)
   - `personality: { tone: 'semiformal', formality: 'voce_sem_girias', humor: 'leve', empathyLevel: 'alta' }`
   - `businessContext: { industry: 'Soluções empresariais', products: 'SaaS de gestão', pricing: 'Sob consulta — nunca inventar valores', policies: 'LGPD-compliant; respeitar opt-out', inviolableRules: ['Nunca prometer preços sem consultar humano', 'Nunca identificar-se como IA a menos que perguntado'] }`
   - `conversationStyle: { greeting: 'cumprimento natural, identifica-se pelo nome', qualificationQuestions: ['Qual o tamanho da empresa?', 'Qual dor principal vocês querem resolver?'], objectionHandling: 'acolher e responder com empatia', handoffTriggers: ['cliente pedir humano', 'pergunta sobre valor específico', 'situação delicada'] }`
   - `status: 'ACTIVE'`
   - `enabledTools: []` (Phase 08 vai popular)
   - `whatsappInstanceId: null` por default — script pede confirmação antes de vincular a uma instância existente
4. Script adicional `link-agent-to-whatsapp.ts` recebe `agentId` + `whatsappInstanceId` via args e atualiza o vínculo
5. Script adicional `enable-ai-on-conversation.ts` recebe `conversationId` + `agentId` e setta `isAIEnabled=true` + `assignedAgentId` — permite Vinni ligar IA numa conversa específica
6. Documentação em `docs/phase-07/seed-testing.md` com:
   - Como rodar cada script
   - Passos do teste manual (ver abaixo nos AC de gate)

### Quality gate — automatizado (@qa)

7. **Typecheck:** `pnpm typecheck` passa em todos os packages sem erro
8. **Lint:** `pnpm lint` passa (warnings permitidos, errors não)
9. **Build:** `pnpm build` passa
10. **Unit tests:** `pnpm test` passa (todos os testes de 07A.4, 07A.5, 07A.6, 07A.7)
11. **Boot check:**
    - `pnpm dev` inicia sem erro
    - Worker inicia sem erro (dev inline OU separado)
    - Mastra Studio acessível em `localhost:4111`
    - Bull-Board acessível em `localhost:3000/admin/queues` (logado como superadmin)
12. **Health endpoints:** todos retornam `status: 'healthy'`:
    - `curl localhost:3000/api/admin/health/queue`
    - `curl localhost:3000/api/admin/health/mastra`
    - `curl localhost:3000/api/admin/health/redis`
    - `curl localhost:3000/api/admin/health/database`
13. **Integration test (opcional mas recomendado):** script que enfileira manualmente um job fake e verifica que worker processa + outbound message é criada com status coerente (pode mockar sender WhatsApp)

### Quality gate — manual (Vinni via documento)

Documentado em `docs/phase-07/gate-07a-manual-checklist.md` pra Vinni seguir:

14. Rodar seed: `pnpm tsx packages/database/seeds/07a-agent-commercial.ts`
15. Rodar link: `pnpm tsx packages/database/seeds/link-agent-to-whatsapp.ts <agentId> <instanceId>`
16. Na UI do chat (Phase 05/06), abrir uma conversa com contato de teste
17. Rodar enable: `pnpm tsx packages/database/seeds/enable-ai-on-conversation.ts <conversationId> <agentId>`
18. Do celular pessoal, enviar mensagem pro número vinculado ao WhatsApp de teste
19. **Verificar:**
    - [ ] Mensagem aparece na conversa (inbound)
    - [ ] Agente responde em português coerente em ≤ 15s (sem tools, só conversa)
    - [ ] Resposta é concisa (1-3 frases, padrão WhatsApp)
    - [ ] Enviar 2ª mensagem: agente lembra do contexto da 1ª
    - [ ] Bull-Board mostra job completed
    - [ ] Health `/admin/health` tudo verde
20. **Teste de restart:**
    - Enviar mensagem
    - Imediatamente: `docker restart vertech-worker` (ou Ctrl+C + restart pnpm dev)
    - Verificar que agente reprocessa e responde (retry automático do BullMQ)
21. **Teste de idempotência:**
    - Simular envio duplicado da mesma mensagem no webhook
    - Verificar que só 1 resposta é gerada

### Relatório QA

22. Relatório em `docs/qa/phase-07a-gate-report.md` com checklist completo, screenshots do Bull-Board, logs relevantes, decisão final: `PASS | CONCERNS | FAIL`
23. Se `CONCERNS`: listar ajustes necessários não-críticos pra serem feitos antes de 07B
24. Se `FAIL`: descrever o que quebrou + @dev re-executa subtasks específicas
