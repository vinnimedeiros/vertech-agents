# Story 07B.1 — Lista de agentes + Novo agente (form MVP)

**Status:** Ready
**Phase:** 07B (UI essencial)
**Owner agent:** `@dev` (Neo)
**Story type:** UI + server actions
**Estimated effort:** 60-90 min
**Validation:** Vinni dispensou `@po *validate-story-draft` em 2026-04-19 (pattern 07A). Story aprovada direto pra `@dev`.

## Context

Substituir os dois placeholders existentes (`/agents` e `/agents/new`) por versões funcionais: (a) lista em grid mostrando os agentes da org, (b) form MVP de criação que cria agente em `DRAFT` e redireciona pro detalhe.

Esta story **não** implementa o detalhe do agente — isso é 07B.2. Aqui foco é: chegar no detalhe com um agente existente.

**Spec de referência:** `docs/phase-07/ui-spec-07b-agent-detail.md` — seção 6 (Tela de lista) e subseção 6.4 (form MVP).

## Acceptance Criteria

### Lista de agentes (`/app/[orgSlug]/agents`)

1. Substituir `ComingSoon` pela lista real respeitando `PageHeader` e botão "+ Novo agente" já existentes no header da rota.
2. Buscar agentes da org ativa via `getAgentsByOrg(organizationId)` (server component, `cache()`).
3. Renderizar grid responsivo: `grid-cols-1 md:grid-cols-2 xl:grid-cols-3 gap-4`.
4. Cada card mostra: avatar 48px, nome (bold, truncate), role (muted, truncate), status badge, nome do modelo (texto curto tipo "GPT-4.1 mini"), vínculo WhatsApp (ou "Sem WhatsApp" se null), versão (ex "v3"), dropdown de ações `⋯`.
5. Click no card (não no dropdown) navega pra `/agents/[agentId]` (aba Identidade — 07B.3 entrega conteúdo).
6. Dropdown `⋯` (usando `DropdownMenu` do shadcn) com itens:
   - Ver detalhe
   - Duplicar (cria cópia em DRAFT com `" (cópia)"` no nome → redireciona pro detalhe da cópia)
   - Ativar / Pausar (dependendo do status atual)
   - Arquivar (abre `AlertDialog` de confirmação)
7. Status badges com cores:
   - `DRAFT` → neutro/cinza
   - `ACTIVE` → verde
   - `PAUSED` → amarelo
   - `ARCHIVED` → vermelho suave
8. Empty state quando sem agentes: usar `ComingSoon` existente com texto atualizado + botão "Criar agente" centralizado (leva pra `/agents/new`):
   - Título: "Seus agentes aparecerão aqui"
   - Descrição: "Crie seu primeiro agente comercial com o formulário rápido."
9. Arquivados **não aparecem** na lista por default. (Opcional 07B: não implementar toggle "ver arquivados" — fica pra 07C.)

### Form MVP (`/app/[orgSlug]/agents/new`)

10. Substituir `ComingSoon` pelo form de criação.
11. Card centralizado `max-w-lg` com `PageHeader` atualizado:
    - Título: "Novo agente"
    - Subtítulo: "Configure o básico — você edita os detalhes depois"
12. Campos:
    - **Nome** (`name`) — `Input`, obrigatório, 2-80 chars
    - **Função** (`role`) — `Input`, opcional, max 80 chars, placeholder "Atendimento comercial"
    - **Modelo** (`model`) — `Select`, obrigatório, default `openai/gpt-4.1-mini`
13. Select de modelo usa lista curada definida em novo arquivo `packages/ai/src/mastra/models.ts` (ou `packages/ai/src/models.ts`), export nomeado `SUPPORTED_MODELS` com estrutura:
    ```typescript
    export const SUPPORTED_MODELS = [
      { id: 'openai/gpt-4.1-mini', label: 'GPT-4.1 mini', provider: 'openai', isDefault: true },
      { id: 'openai/gpt-4.1', label: 'GPT-4.1', provider: 'openai' },
      { id: 'openai/gpt-4o-mini', label: 'GPT-4o mini', provider: 'openai' },
      { id: 'anthropic/claude-haiku-4-5', label: 'Claude Haiku 4.5', provider: 'anthropic' },
      { id: 'anthropic/claude-sonnet-4-6', label: 'Claude Sonnet 4.6', provider: 'anthropic' },
    ];
    ```
14. Select agrupa por provider (OpenAI / Anthropic) visualmente.
15. Botões do rodapé do form:
    - **Cancelar** → `Link` pra `/app/[orgSlug]/agents`
    - **Criar agente** → submit. Disabled se `!form.formState.isValid || isSubmitting`. Spinner + "Criando..." durante submit.
16. Server action `createAgent({ name, role, model, organizationId })`:
    - Valida com zod.
    - Cria em `status: DRAFT`, `version: 1`, defaults pra persona/business/conversationStyle (objetos `{}`), `enabledTools: []`, `knowledgeDocIds: []`, `whatsappInstanceId: null`.
    - RLS: confirma que user atual tem role `admin` ou `owner` na org (retorna 403 caso contrário).
    - Retorna `{ agentId }`, client redireciona via `router.push('/app/[orgSlug]/agents/' + agentId)`.
17. Toast de sucesso após criar: "Agente criado".

### Ações secundárias

18. Server action `duplicateAgent({ agentId })` — cria cópia idêntica com:
    - `name: originalName + ' (cópia)'`
    - `status: 'DRAFT'`
    - `version: 1`
    - `whatsappInstanceId: null` (não duplica vínculo)
    - `publishedAt: null`
19. Server action `archiveAgent({ agentId })` / `unarchiveAgent({ agentId })` — altera só `status`. Arquivar libera o vínculo WhatsApp (`whatsappInstanceId: null`).
20. Server action `toggleAgentStatus({ agentId, to: 'ACTIVE' | 'PAUSED' })` — valida pré-condições antes de ativar (nome, role e modelo preenchidos); se faltar campo, retorna erro com mensagem clara pro toast.
