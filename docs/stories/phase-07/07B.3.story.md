# Story 07B.3 — Aba Identidade

**Status:** Draft
**Phase:** 07B (UI essencial)
**Owner agent:** `@dev` (Neo)
**Story type:** UI (form + server action)
**Estimated effort:** 45-60 min

## Context

Primeira aba do detalhe. Substitui o placeholder criado em 07B.2 e implementa o form de identidade do agente: nome, role, avatar, gênero, descrição.

Esta é a **story modelo** — as abas seguintes (07B.4 a 07B.8) seguem o mesmo esqueleto. Se esta aba sai bem, as outras replicam rápido.

**Spec de referência:** `docs/phase-07/ui-spec-07b-agent-detail.md` — seção 7.1 (aba Identidade).

## Acceptance Criteria

### Pré-requisito

1. Instalar componente `radio-group` do shadcn: `pnpm dlx shadcn-ui@latest add radio-group` (ou via MCP `shadcn`). Garantir que o arquivo `apps/web/modules/ui/components/radio-group.tsx` existe após.

### Page `/agents/[agentId]` (raiz do detalhe)

2. Substituir o placeholder temporário da 07B.2 por `IdentityTab` client component.
3. `IdentityTab` recebe `agent` via `useAgent()` do context (setup em 07B.2).
4. Form usa `useAgentForm({ schema: identitySchema, defaultValues: agent, action: updateAgentIdentity })`.

### Campos

5. **Nome** (`name`) — `Input` (obrigatório, 2-80 chars). Label: "Nome". Helper opcional abaixo: "Como o agente se identifica pras pessoas".
6. **Função** (`role`) — `Input` (opcional, max 80 chars). Label: "Função". Placeholder: "Atendimento comercial". Helper: "Ex: SDR, Atendimento comercial, Qualificador".
7. **Avatar** (`avatarUrl`) — componente `AgentAvatarUpload` (adaptar `UserAvatarUpload` existente em `apps/web/modules/saas/settings/components/UserAvatarUpload.tsx`). Upload pro bucket `avatars`, path `avatars/agent-{agentId}.{ext}`, crop 1:1 (usar `CropImageDialog` existente), max 2MB, formatos png/jpg/webp. Após upload, o avatar aparece no header (revalida o layout).
8. **Gênero** (`gender`) — `RadioGroup` 3 opções horizontais:
    - `FEMININE` — "Feminino"
    - `MASCULINE` — "Masculino"
    - `NEUTRAL` — "Neutro"
    Helper: "Usado pra gerar pronomes corretos nas respostas do agente."
9. **Descrição** (`description`) — `Textarea` 4 rows, max 500 chars. Label: "Descrição". Helper: "Um resumo do que o agente faz (opcional, uso interno)". Contador visual opcional: "X / 500".

### Comportamento

10. Form inicializa com valores atuais do agente. `isDirty` só vira true quando user modifica algo.
11. `DirtyStateBanner` aparece/some conforme `isDirty`.
12. Submit:
    - Chama `updateAgentIdentity({ agentId, name, role, avatarUrl, gender, description })`.
    - Action valida zod, UPDATE no DB, `revalidatePath('/app/[orgSlug]/agents/[agentId]', 'layout')` pra atualizar header + menu + qualquer aba.
    - Toast de sucesso: "Identidade atualizada."
    - Reset do form pros novos valores (`reset(data, { keepDirty: false })`).
13. Inline edit do nome no header (implementado em 07B.2) usa **a mesma** action `updateAgentIdentity` com `name` como único campo. Paridade 1:1.
14. Se agente arquivado (`status === 'ARCHIVED'`): todos os campos `disabled`, banner do 07B.2 aparece acima, `DirtyStateBanner` não renderiza.

### Schema zod

15. Definir em `lib/schemas.ts`:
    ```typescript
    export const identitySchema = z.object({
      name: z.string().min(2).max(80),
      role: z.string().max(80).optional().nullable(),
      avatarUrl: z.string().url().optional().nullable(),
      gender: z.enum(['FEMININE', 'MASCULINE', 'NEUTRAL']),
      description: z.string().max(500).optional().nullable(),
    });
    ```

## Notes

- **Bucket `avatars`:** já existe no Supabase (`reference_obsidian_vault.md`). Confirmar que é `public` (usuário tem link direto pra renderizar). Se `private`, usar URL assinada no render.
- **Crop 1:1:** `CropImageDialog` existente já faz. Reusar sem refactor.
- **Dinâmica do avatar no header:** ao salvar, o `revalidatePath` no layout refresca o `agent.avatarUrl` que o header consome via context. Se houver delay perceptível, pode setar local state otimista no form pra header atualizar imediatamente.
- **Pronomes:** a aba não explica como `gender` é usado downstream — fica pra Phase 08 quando instructions usar esse campo. Por ora só persiste.
- **Descrição:** uso interno puro — não vai pro prompt do agente. Isso é diferente da aba Persona/Negócio.
