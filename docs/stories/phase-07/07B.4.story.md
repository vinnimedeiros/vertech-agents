## Context

Segunda aba do detalhe. Controla o tom, formalidade, humor, empatia e as regras invioláveis do agente — inputs que alimentam o `buildInstructions` do Mastra (Phase 07A).

Esta aba introduz dois padrões novos que o shell 07B.2 não cobriu:
- **Slider com labels extremos** (4 sliders)
- **TagList** pra lista editável de strings (regras invioláveis)

Os dois patterns vão ser reutilizados em 07B.6 (TagList) e 07B.7 (Slider), então vale bem arquitetados aqui.

**Spec de referência:** `docs/phase-07/ui-spec-07b-agent-detail.md` — seção 7.2 (aba Persona).

## Acceptance Criteria

### Page `/agents/[agentId]/persona`

1. Criar `apps/web/app/(saas)/app/(organizations)/[organizationSlug]/agents/[agentId]/persona/page.tsx` renderizando `<PersonaTab />`.
2. `PersonaTab` usa `useAgent()` + `useAgentForm({ schema: personaSchema, defaultValues: agent.personality, action: updateAgentPersona })`.
3. Título da aba: "Persona" + subtítulo: "Como o agente soa nas conversas. Define o tom, formalidade, humor e empatia."

### Campos

4. **4 sliders** (componente reutilizável `LabeledSlider`):

    | Field path | Label | Extremo 0 | Extremo 100 | Default |
    |---|---|---|---|---|
    | `personality.tone` | "Tom" | "Sério" | "Descontraído" | 50 |
    | `personality.formality` | "Formalidade" | "Informal" | "Formal" | 50 |
    | `personality.humor` | "Humor" | "Seco" | "Bem-humorado" | 30 |
    | `personality.empathyLevel` | "Empatia" | "Objetivo" | "Acolhedor" | 70 |

5. **`LabeledSlider`** — componente custom reutilizável:
    - `Slider` (shadcn) min=0 max=100 step=1
    - Label acima (bold)
    - Extremo 0 à esquerda e extremo 100 à direita (text-xs muted)
    - Valor atual abaixo do slider como texto curto: calcular a label interpolada. Ex: `tone: 72` → "Descontraído (72%)". `tone: 50` → "Neutro (50%)". A função `describeSliderValue(value, labels)` retorna algo como:
        - 0-20: labels[0] (ex: "Sério")
        - 21-40: "Tendendo a " + labels[0] (ex: "Tendendo a sério")
        - 41-60: "Neutro"
        - 61-80: "Tendendo a " + labels[1] (ex: "Tendendo a descontraído")
        - 81-100: labels[1] (ex: "Descontraído")
    - Sempre mostra `(N%)` após a label.

6. **Regras invioláveis** (`personality.inviolableRules`) — componente `TagList`:
    - Lista de strings editáveis, cada uma max 80 chars, array max 20 items.
    - UI: lista vertical. Cada item é um "chip" retangular (não redondo) com texto + botão `X` à direita pra remover.
    - Input + botão "Adicionar" no final da lista. Enter no input também adiciona.
    - Helper: "Regras que o agente NUNCA deve quebrar. Ex: 'Nunca prometer preços sem consultar humano', 'Nunca identificar-se como IA a menos que perguntado'."

### Comportamento

7. `DirtyStateBanner` renderiza no rodapé.
8. Submit:
    - `updateAgentPersona({ agentId, personality })` onde `personality` é o JSONB completo.
    - Toast: "Persona atualizada."
    - `revalidatePath` no layout do detalhe.
9. Reset pros valores atuais funciona mesmo com TagList (regras voltam ao estado inicial).
10. Validação antes do submit: sliders são integers 0-100, `inviolableRules` array max 20 strings max 80 chars.
11. Se agente arquivado: todos os controles `disabled`.

### Schema zod

12. Adicionar em `lib/schemas.ts`:
    ```typescript
    export const personalitySchema = z.object({
      tone: z.number().int().min(0).max(100),
      formality: z.number().int().min(0).max(100),
      humor: z.number().int().min(0).max(100),
      empathyLevel: z.number().int().min(0).max(100),
      inviolableRules: z.array(z.string().min(1).max(80)).max(20),
    });
    export const personaSchema = z.object({
      personality: personalitySchema,
    });
    ```
