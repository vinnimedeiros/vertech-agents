# Story 07B.5 — Aba Negócio

**Status:** Draft
**Phase:** 07B (UI essencial)
**Owner agent:** `@dev` (Neo)
**Story type:** UI (form com textareas)
**Estimated effort:** 30-45 min

## Context

Terceira aba. Configura o contexto de negócio que alimenta o `buildInstructions` — o que o agente sabe sobre indústria, produtos, pricing e políticas.

Esta é a aba mais simples estruturalmente: 4 textareas + 1 botão inline (reaproveita TagList da 07B.4, se aplicável). Nenhum pattern novo — é colar os tijolos.

**Spec de referência:** `docs/phase-07/ui-spec-07b-agent-detail.md` — seção 7.3 (aba Negócio).

## Acceptance Criteria

### Page `/agents/[agentId]/business`

1. Criar `apps/web/app/(saas)/app/(organizations)/[organizationSlug]/agents/[agentId]/business/page.tsx` renderizando `<BusinessTab />`.
2. `BusinessTab` usa `useAgent()` + `useAgentForm({ schema: businessSchema, defaultValues: agent.businessContext, action: updateAgentBusinessContext })`.
3. Título: "Negócio" + subtítulo: "O que o agente sabe sobre o seu negócio — indústria, produtos, políticas."

### Campos

4. **Indústria** (`businessContext.industry`) — `Input`, max 80 chars. Helper: "Ex: Soluções empresariais, SaaS B2B, Consultoria financeira".
5. **Produtos e serviços** (`businessContext.products`) — `Textarea` 6 rows, max 2000 chars. Helper: "Liste o que o agente pode oferecer. Ex: 'Plataforma de gestão — planos Basic (R$ sob consulta), Pro e Enterprise'". Contador "X / 2000" visual opcional.
6. **Política de preços** (`businessContext.pricing`) — `Textarea` 3 rows, max 500 chars. Helper: "Como o agente trata perguntas sobre valor. Ex: 'Sob consulta — nunca inventar valores específicos'".
7. **Políticas importantes** (`businessContext.policies`) — `Textarea` 4 rows, max 1000 chars. Helper: "Regras importantes do negócio. Ex: 'LGPD-compliant; respeitar opt-out; disclaimer obrigatório em primeira mensagem'".

### Comportamento

8. `DirtyStateBanner` no rodapé.
9. Submit:
    - `updateAgentBusinessContext({ agentId, businessContext })` onde `businessContext` é o JSONB.
    - Toast: "Contexto de negócio atualizado."
    - `revalidatePath` do detalhe.
10. Estado arquivado bloqueia edição.

### Schema zod

11. Adicionar em `lib/schemas.ts`:
    ```typescript
    export const businessContextSchema = z.object({
      industry: z.string().max(80).optional().nullable(),
      products: z.string().max(2000).optional().nullable(),
      pricing: z.string().max(500).optional().nullable(),
      policies: z.string().max(1000).optional().nullable(),
    });
    export const businessSchema = z.object({
      businessContext: businessContextSchema,
    });
    ```

## Notes

- **Contador de chars:** use um helper `<CharCount value={fieldValue} max={2000} />` pra reuso (aparece abaixo do campo à direita, em text-xs muted).
- **Campos nullable:** zod aceita `.optional().nullable()` mas react-hook-form trata empty string como "" não como null. Ao submit, converter "" para null antes de mandar pra action pra manter consistência no DB.
- **Expansível?** Textareas podem ser `resize-none` pra não quebrar layout, ou `resize-y` controlado. **Padrão:** `resize-none` mantendo rows fixos — mais estável visualmente.
