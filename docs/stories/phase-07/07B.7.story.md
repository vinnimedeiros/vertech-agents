# Story 07B.7 — Aba Modelo

**Status:** Draft
**Phase:** 07B (UI essencial)
**Owner agent:** `@dev` (Neo)
**Story type:** UI (form com radio + select + sliders)
**Estimated effort:** 45-60 min

## Context

Quinta aba. Escolhe o LLM (OpenAI / Anthropic) e ajusta temperatura + máximo de passos. Muda diretamente como o agente responde (criativo vs preciso, quantas tools pode encadear).

Reutiliza `LabeledSlider` da 07B.4. Reutiliza `SUPPORTED_MODELS` da 07B.1.

**Spec de referência:** `docs/phase-07/ui-spec-07b-agent-detail.md` — seção 7.5 (aba Modelo).

## Acceptance Criteria

### Page `/agents/[agentId]/model`

1. Criar `apps/web/app/(saas)/app/(organizations)/[organizationSlug]/agents/[agentId]/model/page.tsx` renderizando `<ModelTab />`.
2. `ModelTab` usa `useAgent()` + `useAgentForm({ schema: modelSchema, defaultValues: { model, temperature, maxSteps }, action: updateAgentModel })`.
3. Título: "Modelo" + subtítulo: "Escolha o cérebro do agente e ajuste o estilo de resposta."

### Campos

4. **Provider** (UI-only, derivado de `model`) — `RadioGroup` 2 opções horizontais:
    - `openai` — "OpenAI" (ícone `openai` simples ou letra `O`)
    - `anthropic` — "Anthropic" (letra `A`)
    Ao mudar provider, `model` é setado pro primeiro da lista `SUPPORTED_MODELS` filtrada por provider (use o que tem `isDefault: true` do provider, ou o primeiro).
5. **Modelo** (`model`) — `Select` filtrado por provider atual. Opções vêm de `SUPPORTED_MODELS.filter(m => m.provider === selectedProvider)`. Mostra `label` (ex: "GPT-4.1 mini") com `id` como `value`. Agrupamento visual opcional (shadcn `Select` suporta labels).
6. **Temperatura** (`temperature`) — `LabeledSlider` min=0 max=2 step=0.1, default 0.7. Labels extremos: "Preciso (0.0)" / "Criativo (2.0)". Value description: mostra `value.toFixed(1)` fixo. Helper: "Quanto maior, mais variadas e criativas as respostas. Quanto menor, mais direto e previsível."
7. **Máximo de passos** (`maxSteps`) — `LabeledSlider` min=1 max=20 step=1, default 10. Labels extremos: "1 passo" / "20 passos". Value description: mostra o valor inteiro. Helper: "Quantas tools o agente pode encadear numa mesma resposta (deixe em 10 na dúvida)."

### Comportamento

8. `DirtyStateBanner` no rodapé.
9. Submit:
    - `updateAgentModel({ agentId, model, temperature, maxSteps })`.
    - Toast: "Modelo atualizado."
    - `revalidatePath` do detalhe (atualiza subtítulo do header que mostra o modelo).
10. Provider muda → `model` muda automaticamente → form vira dirty (mesmo só mudando provider).
11. **Validação cross-field:** não é estritamente necessário validar que `model` existe em `SUPPORTED_MODELS` porque só os IDs são selecionáveis. Mas zod pode validar via enum pra segurança:
    ```typescript
    model: z.enum([
      'openai/gpt-4.1-mini',
      'openai/gpt-4.1',
      'openai/gpt-4o-mini',
      'anthropic/claude-haiku-4-5',
      'anthropic/claude-sonnet-4-6',
    ]),
    ```
    Ou derivar dinamicamente: `z.string().refine(v => SUPPORTED_MODELS.some(m => m.id === v), 'Modelo inválido')`.
12. Estado arquivado bloqueia edição.

### Schema zod

13. Adicionar em `lib/schemas.ts`:
    ```typescript
    export const modelSchema = z.object({
      model: z.string().refine(v => SUPPORTED_MODELS.some(m => m.id === v), {
        message: 'Modelo não suportado',
      }),
      temperature: z.number().min(0).max(2),
      maxSteps: z.number().int().min(1).max(20),
    });
    ```

### LabeledSlider suporte a `step` e valor com casa decimal

14. Verificar que `LabeledSlider` (criado em 07B.4) aceita `step` e lida com valor float. Se não, ajustar: aceitar prop `step: number` e prop `formatValue: (v) => string` (default `(v) => v.toString()`). Na aba Persona o formatValue é integer, aqui é `v.toFixed(1)`.

## Notes

- **Provider como UI-only:** não persistido separado. Derivado no render via `SUPPORTED_MODELS.find(m => m.id === form.watch('model'))?.provider`.
- **Controlled RadioGroup:** `value={provider}` derivado. `onValueChange={(newProvider) => { const firstModel = SUPPORTED_MODELS.find(m => m.provider === newProvider && m.isDefault) ?? SUPPORTED_MODELS.find(m => m.provider === newProvider); form.setValue('model', firstModel!.id, { shouldDirty: true }); }}`.
- **Modelos adicionais no futuro:** quando Phase 08+ adicionar novos modelos, só atualizar `SUPPORTED_MODELS`. Sem mudar esta aba.
- **Slider temperature com step 0.1:** Radix Slider aceita floats. Formatar com `.toFixed(1)` no display. Persistir como float no DB.
