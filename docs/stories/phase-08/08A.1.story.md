# Story 08A.1 — Package rag infrastructure (ingest + extractors + query)

**Status:** Draft
**Phase:** 08-alpha
**Owner agent:** `@dev` (Neo)
**Story type:** Backend package + extractors + chunking + embedding + retrieval
**Estimated effort:** 4-6h

## Context

Criar o módulo `packages/ai/src/rag/` que é a espinha dorsal de retrieval-augmented generation do Vertech. Todas as outras stories de 08-alpha e 09 dependem desta. Entrega:

1. Extractors de texto por tipo de arquivo (PDF, DOCX, CSV, XLSX, TXT, URL).
2. Pipeline de ingest completo: baixa arquivo do Storage, extrai texto, chunka via `@mastra/rag` (recursive, size 512, overlap 50), gera embeddings via `embedMany` + OpenAI `text-embedding-3-small` (1536d), upserta em `knowledge_chunks` via `@mastra/pg` `PgVector`.
3. Função `queryKnowledge(sessionId | agentId, query, topK)` pra retrieval com `SET LOCAL hnsw.ef_search = 100` e filter por scope.
4. Função auxiliar `generateDocSummary` pra resumir primeiros ~3000 chars do doc (usado no recap da etapa Conhecimento do Arquiteto).

**Specs de referência:**
- Tech Spec § 5 (RAG strategy): `docs/phase-09/tech-spec-arquiteto.md`
- Research § 2 + § 3: `docs/phase-09/research-dependencies.md`

## Acceptance Criteria

### Setup e dependencies

1. GIVEN projeto está em `packages/ai`, WHEN rodar `pnpm add pdf-parse mammoth papaparse xlsx cheerio @mastra/rag @mastra/pg`, THEN instalação sucede sem conflito.
2. GIVEN `@mastra/rag` instalado, WHEN importar `MDocument` de `@mastra/rag`, THEN type-check passa.

### Extractors (packages/ai/src/rag/extractors/)

3. GIVEN buffer de PDF 10 páginas, WHEN chamar `extractPdf(buffer)`, THEN retorna string com texto concatenado por página.
4. GIVEN buffer de DOCX, WHEN chamar `extractDocx(buffer)`, THEN retorna texto limpo (sem tags XML residuais).
5. GIVEN buffer de CSV, WHEN chamar `extractCsv(buffer)`, THEN retorna texto estruturado (headers + linhas separadas por `\n`).
6. GIVEN buffer de XLSX, WHEN chamar `extractXlsx(buffer)`, THEN retorna texto com nome da sheet + conteúdo.
7. GIVEN buffer de TXT, WHEN chamar `extractTxt(buffer)`, THEN retorna string UTF-8.
8. GIVEN URL válida, WHEN chamar `extractUrl(url)`, THEN faz fetch, usa `cheerio` pra extrair texto visível, retorna string.
9. GIVEN buffer inválido/corrompido, WHEN chamar qualquer extractor, THEN lança `ExtractorError` com mensagem clara (não silencia).

### Ingest pipeline (packages/ai/src/rag/ingest.ts)

10. GIVEN `documentId` válido com status PENDING, WHEN chamar `ingestDocument(documentId)`, THEN: atualiza status → PROCESSING, baixa file do Supabase Storage, roda extractor, chunka com `MDocument.fromText(text).chunk({ strategy: 'recursive', size: 512, overlap: 50, separators: ['\n\n','\n','. ',' '] })`, chama `embedMany` com `ModelRouterEmbeddingModel('openai/text-embedding-3-small')`, upserta em `knowledge_chunks` via `PgVector.upsert`, gera summary dos primeiros 3000 chars, atualiza `status: READY, chunkCount, extractedSummary`, emite evento `knowledge.document.ready`.
11. GIVEN qualquer falha no pipeline, WHEN exceção ocorre, THEN seta `status: ERROR, errorMessage`, emite evento `knowledge.document.failed`, NÃO rejoga exceção pra queue retry automático (Neo decide strategy de retry).
12. GIVEN document com `sessionId` não-null, WHEN ingesta, THEN metadata de cada chunk inclui `{ text, documentTitle, position, totalChunks, sessionId }`.
13. GIVEN document com `agentId` não-null (pós-publish), WHEN ingesta, THEN metadata inclui `{ ..., agentId }`.

### Query / Retrieval (packages/ai/src/rag/query.ts)

14. GIVEN `queryKnowledge(sessionId, queryString, topK=5, threshold=0.5)`, WHEN chamar, THEN: gera embedding do queryString, roda `SET LOCAL hnsw.ef_search = 100`, chama `pgVector.query({ indexName: 'knowledge_chunks', queryVector, topK: topK*2, filter: { sessionId } })`, filtra por `score > threshold`, retorna top-K chunks.
15. GIVEN `queryKnowledge(null, queryString, ...)` com `agentId` não-null em vez de sessionId, WHEN chamar, THEN filter é `{ agentId }`.
16. GIVEN retrieval sem resultados acima do threshold, WHEN chamar, THEN retorna array vazio (não erro).

### Summary helper (packages/ai/src/rag/summary.ts)

17. GIVEN texto de 5000+ chars, WHEN chamar `generateDocSummary(text)`, THEN retorna string de 100-200 chars resumindo conteúdo principal. Pode usar LLM (OpenAI gpt-4o-mini) ou heurística (primeiros N chars tratados).

### Quality

18. `pnpm -F @repo/ai type-check` passa.
19. Testes unitários pros extractors (happy path por tipo).

## Dev Agent Record (Neo)

### Divergências da spec da Aria (técnicas, documentadas)

1. **API @mastra/rag v2**: usa `maxSize` em vez de `size` no `chunk()`. Atualizei pra `maxSize: 512`.
2. **Embedder**: spec da Aria usava `ModelRouterEmbeddingModel` (Mastra v2, spec v2). Isso é incompatível com `embedMany` do pacote `ai` v4 (spec v1). Troquei por `openai.embedding('text-embedding-3-small')` do `@ai-sdk/openai` direto (spec v1). Resultado final idêntico: OpenAI text-embedding-3-small 1536d.
3. **pdf-parse v2**: API mudou, não tem `default` export. Usei fallback com cast `default ?? pdf` pra compatibilidade.
4. **PgVector SET LOCAL**: `SET LOCAL` requer transação ativa e PgVector usa pool próprio. Implementei fallback pra `SET` session-level + silencioso em caso de falha (query ainda retorna resultados com ef_search default 40).
