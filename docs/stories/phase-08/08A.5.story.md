## Acceptance Criteria

### Validações de pipeline RAG (end-to-end real)

1. GIVEN PDF de 5 páginas (tabela de preços clínica, ~3KB texto), WHEN upload via `/api/architect/upload` mockando sessão, THEN:
   - Document aparece em `knowledge_document` status=PENDING dentro de 1s
   - Job enqueued em `ingest-document` queue
   - Worker processa em <30s
   - Chunks aparecem em `knowledge_chunks` com embeddings preenchidos
   - `chunkCount` coerente (pdf com ~3KB gera ~10-15 chunks com size=512 overlap=50)
   - `extractedSummary` preenchido (100-200 chars)
   - `status: READY` final
   - Evento `knowledge.document.ready` emitido em Supabase Realtime

2. GIVEN query `"qual o valor do clareamento?"` via `queryKnowledge(sessionId, query)`, WHEN chunks indexados do PDF, THEN:
   - Retorna top-5 chunks ordenados por cosine similarity
   - Top-1 contém informação de preço do clareamento (similarity >0.6)
   - Latência p95 <100ms

3. GIVEN documento corrompido (PDF malformado), WHEN upload + ingest, THEN:
   - `status: ERROR` com `errorMessage` descritiva
   - Evento `knowledge.document.failed` emitido
   - Job não retenta infinitamente (max 3 attempts)

### Validações de architectTools (via unit + integration tests)

4. GIVEN cenário `generateArtifact('business_profile')` com working memory checklist.ideation incompleto, WHEN execute, THEN retorna erro `CHECKLIST_INCOMPLETE` com lista de campos null.

5. GIVEN artefato `business_profile` gerado + aprovado, WHEN `refineArtifact(id, 'adicionar serviço X')`, THEN retorna erro `ARTIFACT_LOCKED` (artefato status=approved trava refinamento).

6. GIVEN `publishAgentFromSession(sessionId)` com checklist completo, WHEN execute em sessão de teste, THEN:
   - Cria agent com status=DRAFT, version=1
   - knowledge_documents migrados (agentId preenchido, sessionId null)
   - Metadata dos chunks atualizada
   - agent_version v1 criada com snapshot
   - agent_creation_session.status=PUBLISHED
   - orchestrator_audit_log entry com actorType='architect'

7. GIVEN `publishAgentFromSession` com falha no middle (ex: mock force error no step 5), WHEN transação rollback, THEN:
   - Nenhum agent criado
   - Nenhum knowledge_document migrado
   - Session permanece DRAFT
   - Nenhum audit log criado

8. GIVEN 2 updates concurrent em `refineArtifact` (race), WHEN optimistic lock verifica `version`, THEN uma falha com `CONCURRENT_UPDATE`, outra sucede.

### Validações de RLS

9. GIVEN user A (owner org X) + knowledge_document da org X, WHEN query com session user A, THEN vê.

10. GIVEN user B (owner org Y) + knowledge_document da org X, WHEN query com session user B, THEN RLS bloqueia (retorna vazio).

11. GIVEN user member (role=member na org X), WHEN INSERT em knowledge_document da org X, THEN RLS bloqueia (só admin/owner insere).

12. GIVEN agent_creation_session do user A, WHEN user B tenta SELECT, THEN RLS bloqueia (só dono vê, exceto admin da org).

### Validações técnicas

13. `pnpm -F @repo/ai type-check` passa.
14. `pnpm -F @repo/queue type-check` passa.
15. `pnpm -F web type-check` passa (endpoints /api/architect/upload compilam).
16. `pnpm -r lint` passa (sem erros novos).
17. `pnpm -r build` passa.

### Validações de performance

18. GIVEN ~200 chunks indexados, WHEN `queryKnowledge` com `SET LOCAL hnsw.ef_search = 100`, THEN p95 latência <100ms (medido em 10 queries diferentes).
19. GIVEN upload PDF 5MB, WHEN ingest end-to-end, THEN <60s (p95).
