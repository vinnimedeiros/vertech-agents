## Acceptance Criteria

### Estado 1: Sessão retomada (após close/reopen)

1. GIVEN user criou rascunho "Sofia Clínica", fechou aba, WHEN volta a `/agents/new?session={sessionId}` horas depois, THEN: spinner "Retomando sessão..." breve, chat carrega histórico, primeira mensagem nova do Arquiteto cumprimenta com contexto e 2 botões quick-action: `[↪ Continuar planejamento]` / `[🔍 Revisar tudo]`.
2. GIVEN click "Continuar planejamento", WHEN callback, THEN foca composer, Arquiteto faz próxima pergunta pendente.
3. GIVEN click "Revisar tudo", WHEN callback, THEN scroll pro topo do chat, mostra todos artefatos aprovados.

### Estado 2: Erro de publicação

4. GIVEN user clica "Criar agente" (09.9), WHEN server action falha (ex: DB transient error), THEN Arquiteto envia mensagem: "Ops, tive um problema salvando. Seus dados estão seguros. Vamos tentar de novo?" + botão `[Tentar criar novamente]`.
5. GIVEN 3 retries falhos, WHEN persistente, THEN mostra código `ERR-{sessionId}` + orientação "Anote esse código pra suporte".

### Estado 3: Char limit atingido

6. GIVEN textarea com chars se aproximando do limite 4000, WHEN renderiza counter, THEN thresholds: 3500-3899 muted, 3900-3999 amber, >=4000 destructive + botão enviar disabled + tooltip "Mensagem muito longa, divide em partes".

### Estado 4: Offline

7. GIVEN `navigator.onLine === false`, WHEN detectado, THEN header mostra badge vermelho pulsante "● Offline", composer disabled com label "Sem conexão. Suas mensagens vão ser enviadas quando voltar.".
8. GIVEN user digita offline, WHEN submete, THEN mensagem fica em queue local (`localStorage` key `architect_queue_{sessionId}`).
9. GIVEN online volta, WHEN reconecta, THEN toast "Conexão restaurada. Enviando 2 mensagens pendentes...", queue processa em ordem.

### Estado 5: Rate limit

10. GIVEN user excede 10 msg/min, WHEN backend retorna 429, THEN toast warning "Você está falando muito rápido. Arquiteto precisa de tempo pra processar. Tente novamente em {ttl}s.".
11. GIVEN rate limit ativo, WHEN composer, THEN disabled com countdown visual até TTL expirar.

### E2E tests (Playwright)

12. GIVEN teste E2E "criar agente end-to-end" (`apps/web/tests/e2e/architect-create-agent.spec.ts`), WHEN Oracle roda, THEN:
    - Navigate /agents, click "Criar com o Arquiteto"
    - Selecionar template "Clínica"
    - Responder 5 perguntas da Ideação
    - Aprovar artefato Perfil do Negócio
    - Responder 3 perguntas do Planejamento
    - Refinar Blueprint via Dialog (mudar 1 slider + 1 anti-pattern)
    - Aprovar Blueprint
    - Upload de 1 PDF (mock)
    - Aprovar Conhecimento
    - Ver Resumo Final + Flow Diagram
    - Clicar "Criar agente"
    - Verify redirect pra `/agents/[agentId]`
    - Verify agent criado no DB com status DRAFT

13. GIVEN teste E2E "retomar sessão" (`architect-resume-session.spec.ts`), WHEN rodado, THEN:
    - Criar rascunho até etapa Planejamento
    - Fechar aba
    - Reabrir `/agents/new?session=...`
    - Verify chat carrega histórico
    - Verify Arquiteto cumprimenta com contexto

14. GIVEN teste E2E "upload + refino via chat" (`architect-upload-refine.spec.ts`), WHEN rodado, THEN:
    - Criar rascunho
    - Upload PDF mid-conversa
    - Verify mini-card aparece e muda pra indexed
    - Verify Arquiteto acusa recebimento
    - Quando Planejamento card gerado, clicar "Mandar alteração no chat"
    - Digitar instrução natural "deixa mais formal"
    - Verify card regenera e atualiza

### Métricas de sucesso (PRD v2 § 10)

15. GIVEN time-to-first-active-agent, WHEN medido em teste E2E + real user, THEN < 30 min (sem gate bloqueante, mas métrica reportada).
16. GIVEN latência do Arquiteto por turno, WHEN telemetria simples em dev, THEN p50 < 5s (mais permissivo que PRD v2 <3s pra beta).

### QA gate report

17. GIVEN `docs/qa/phase-09-gate-report.md` (Oracle cria), WHEN gate, THEN verdict PASS/CONCERNS/FAIL com issues detalhados.

### Gate humano Vinni

18. GIVEN Oracle entrega PASS ou CONCERNS resolvível, WHEN Vinni recebe, THEN valida manualmente:
    - Criar um agente odontológico do zero (tempo ~15-20 min)
    - Refinar artefato via Dialog
    - Refinar via chat
    - Upload de 1 PDF de teste
    - Sandbox NÃO precisa funcionar ainda (é 07B-v2)
    - Publicar agente, ver redirect
