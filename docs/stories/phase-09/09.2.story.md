# Story 09.2 — Shell do chat `/agents/new` (Header + StatusBar + área de mensagens)

**Status:** Draft
**Phase:** 09
**Owner agent:** `@dev` (Neo)
**Story type:** UI shell + layout structure
**Estimated effort:** 3-4h

## Context

Cria o shell visual e layout da tela de chat do Arquiteto. Esta story entrega **só a estrutura** (header fino, status-bar 4 etapas, container de mensagens vazias com scroll). Lógica de chat real, composer, anexos e artefatos vêm nas stories seguintes.

É a base sobre a qual tudo mais é montado.

**Specs de referência:**
- UI Spec § 4 (layout 4 zonas): `docs/phase-09/ui-spec-arquiteto.md`
- UI Spec § 4.2-4.6 (anatomia zona por zona)

## Acceptance Criteria

### Route e layout

1. GIVEN rota `/app/[orgSlug]/agents/new`, WHEN acessada sem query params, THEN redireciona pra `/app/[orgSlug]/agents` (tela de boas-vindas).
2. GIVEN rota com `?template={id}`, WHEN `templateId` não existe em registry, THEN 404.
3. GIVEN rota com `?session={sessionId}`, WHEN session não pertence ao user ou não existe, THEN 404.
4. GIVEN rota válida, WHEN renderiza, THEN layout vertical em 4 zonas conforme UI Spec § 4.2.

### Header fino (zona 1, altura 48px)

5. GIVEN header, WHEN renderiza, THEN esquerda: `ArrowLeft` + breadcrumb "{templateLabel} ▸ Novo agente".
6. GIVEN click em breadcrumb template, WHEN navega, THEN vai pra `/agents` (tela de boas-vindas).
7. GIVEN click no botão voltar, WHEN form está dirty (tem session ativa com msgs), THEN abre `AlertDialog`: "Você tem mudanças não salvas..." com [Continuar aqui] e [Salvar rascunho e sair].
8. GIVEN botão "Salvar e sair" à direita, WHEN click, THEN mesmo comportamento do AlertDialog (save então volta pra /agents).

### Status-bar fino (zona 2, altura 28px)

9. GIVEN status-bar, WHEN renderiza, THEN 4 etapas: "Ideação", "Planejamento", "Conhecimento", "Criação".
10. GIVEN estado pendente, WHEN renderiza etapa, THEN dot `○` oco muted.
11. GIVEN estado ativo, WHEN renderiza, THEN dot `●` preenchido primary + texto primary font-medium.
12. GIVEN estado concluído, WHEN renderiza, THEN check verde `✓`.
13. GIVEN mobile (<768px), WHEN responsivo, THEN mostra só dots + label da etapa atual com "({n}/4)".
14. GIVEN componente, WHEN acessível, THEN `role="status"` + `aria-live="polite"` anuncia mudanças.

### Área de mensagens (zona 3)

15. GIVEN container, WHEN renderiza, THEN `max-w-[800px] mx-auto px-6` centralizado, flex-1 ocupa espaço entre status-bar e composer, scroll vertical suave.
16. GIVEN padding inferior, WHEN scroll no fundo, THEN msgs não ficam coladas no composer (pb-12).
17. GIVEN estado vazio (primeira msg do Arquiteto ainda não chegou), WHEN carrega, THEN mostra shimmer/placeholder de "Arquiteto está preparando...".

### Composer (zona 4, placeholder)

18. GIVEN zona 4, WHEN renderiza, THEN placeholder visual do composer (textarea disabled + botão enviar disabled). Integração funcional vem na 09.3.

### Server component + template loading

19. GIVEN rota renderiza, WHEN carrega, THEN: busca template de `templates.ts` (09.1), busca session se existe, busca user session (auth). Tudo em server component.
20. GIVEN session existe, WHEN render, THEN passa sessionId inicial ao client chat (09.5 consome).

### Responsividade

21. GIVEN desktop, WHEN renderiza, THEN chat container 800px, laterais vazias bg-background.
22. GIVEN tablet, WHEN renderiza, THEN 100% width com padding 32px.
23. GIVEN mobile, WHEN renderiza, THEN 100% com padding 16px, status-bar compacta.

### Translations

24. GIVEN textos novos, WHEN i18n, THEN adicionar `architect.chat.header.*` e `architect.chat.statusBar.*` (chaves completas em UI Spec § 14).
