# Story 09.3 — Composer com textarea expansível + keyboard shortcuts

**Status:** Draft
**Phase:** 09
**Owner agent:** `@dev` (Neo)
**Story type:** UI component + accessibility
**Estimated effort:** 2-3h

## Context

Composer é a área de input do usuário no chat do Arquiteto. Textarea expansível (1→8 rows), botão enviar, atalhos de teclado (Enter envia, Shift+Enter quebra linha, Cmd+K abre menu anexo).

Esta story entrega o composer **sem upload** ainda (upload vem na 09.4). Foco aqui: textarea + enviar + shortcuts + char counter.

**Specs de referência:**
- UI Spec § 4.7 (Composer anatomia): `docs/phase-09/ui-spec-arquiteto.md`
- UI Spec § 10.3 (char counter limits)
- Decisão: SEM voice input (Dilema 5)

## Acceptance Criteria

### Layout base

1. GIVEN composer, WHEN renderiza, THEN container `max-w-[800px] mx-auto` alinhado com área de mensagens, sticky bottom (`position: sticky; bottom: 0`), border-top sutil.
2. GIVEN padding, WHEN layout, THEN 16px em todos os lados.
3. GIVEN layout interno, WHEN renderiza, THEN: botão `Paperclip` à esquerda (disabled na 09.3, funcional na 09.4), textarea central expansível, botão `ArrowUp` à direita.

### Textarea expansível

4. GIVEN textarea, WHEN vazio, THEN `rows={1}`. GIVEN user digita e alcança fim de linha, THEN auto-expand até `rows={8}` (max).
5. GIVEN auto-expand, WHEN ocorre, THEN altura suave (`transition-all duration-150`).
6. GIVEN placeholder default, WHEN vazio e idle, THEN "Digite sua mensagem...".
7. GIVEN placeholder bloqueado (aguardando processar anexos futuros), WHEN disabled, THEN "Aguardando processar anexos...". (Prop `blocked: boolean`.)
8. GIVEN placeholder offline, WHEN `!navigator.onLine`, THEN "Sem conexão. Suas mensagens vão ser enviadas quando voltar.".

### Botão enviar

9. GIVEN textarea vazia, WHEN renderiza, THEN `ArrowUp` disabled.
10. GIVEN textarea com texto, WHEN user digitou, THEN button enabled com `variant="default"`.
11. GIVEN submit em andamento (`isSubmitting: true`), WHEN click ou keyboard, THEN spinner substitui `ArrowUp`, button disabled.

### Keyboard shortcuts

12. GIVEN textarea focada, WHEN user aperta `Enter` sem shift, THEN chama `onSend(text)`, textarea limpa.
13. GIVEN user aperta `Shift+Enter`, WHEN textarea focada, THEN insere quebra de linha.
14. GIVEN user aperta `Cmd/Ctrl+K`, WHEN textarea focada, THEN emite evento `onOpenAttachmentMenu()` (handler provido por parent).
15. GIVEN IME (composição de caracteres em alguns idiomas), WHEN composing ativo, THEN `Enter` NÃO envia (use `e.isComposing`).

### Char counter

16. GIVEN textarea com <3500 chars, WHEN renderiza, THEN sem counter visível.
17. GIVEN 3500-3899 chars, WHEN atinge, THEN counter `3500/4000` aparece em muted.
18. GIVEN 3900-3999 chars, WHEN atinge, THEN counter em amber (warning).
19. GIVEN >=4000 chars, WHEN atinge, THEN counter em destructive red, botão enviar disabled, tooltip "Mensagem muito longa, divide em partes".

### Accessibility

20. GIVEN textarea, WHEN screen reader, THEN `aria-label="Mensagem para o Arquiteto"`.
21. GIVEN composer, WHEN renderiza, THEN focus visible herdado do tema.
22. GIVEN user Tab navega, WHEN focus passa pelos botões, THEN ordem lógica: anexo → textarea → enviar.

### API do componente

23. GIVEN `<ArchitectComposer />`, WHEN props, THEN interface: `{ onSend(text, attachments), onOpenAttachmentMenu(), disabled?, blocked?, maxChars=4000 }`.
24. GIVEN parent controla state, WHEN pai muda prop `disabled`, THEN composer desabilita tudo.

### Translations

25. GIVEN textos, WHEN i18n, THEN adicionar `architect.chat.composer.*` chaves (UI Spec § 14).

## Scope IN

- Textarea auto-expand 1→8 rows
- Botão enviar com states (idle/typing/submitting)
- Keyboard shortcuts (Enter/Shift+Enter/Cmd+K)
- Char counter com 3 thresholds (3500/3900/4000)
- Placeholder dinâmico por estado
- Accessibility completa
- Translations

- Voice input (adiado, fora do MVP)
