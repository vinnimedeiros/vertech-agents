# Story 09.6 — ArtifactCard inline + 3 actions

**Status:** Draft
**Phase:** 09
**Owner agent:** `@dev` (Neo)
**Story type:** UI component + state management
**Estimated effort:** 3-4h

## Context

Cards de artefato são a materialização visível do que o Arquiteto produz em cada etapa. Aparecem inline dentro de mensagens do Arquiteto com 3 ações: Refinar (abre painel na 09.7/9.8), Mandar alteração no chat (foca composer), Aprovar (trava e avança etapa).

Esta story entrega o componente base + integração com chat. Os painéis de refino vêm separados (09.7 inline, 09.8 Dialog).

**Specs de referência:**
- UI Spec § 5 (anatomia card + 3 actions + 4 tipos): `docs/phase-09/ui-spec-arquiteto.md`
- UI Spec § 5.6 (placement no chat)

## Acceptance Criteria

### Componente base

1. GIVEN `ArtifactCard` em `apps/web/modules/saas/agents/architect/components/artifacts/ArtifactCard.tsx`, WHEN recebe props `{ artifact, onRefine, onChatChange, onApprove }`, THEN renderiza card com título + divisor + conteúdo estruturado + footer de ações.
2. GIVEN card, WHEN renderiza, THEN styles: `bg-card border border-border rounded-lg p-5 max-w-[640px]`.
3. GIVEN título, WHEN renderiza, THEN `font-semibold text-base` com emoji embutido (`📋`) + label do tipo (ex: "Perfil do Negócio").

### Renderização do content por tipo

4. GIVEN `artifact.type === 'business_profile'`, WHEN renderiza, THEN mostra: Nome do negócio, Resumo, Serviços (lista), Público, Objetivo do agente.
5. GIVEN `artifact.type === 'agent_blueprint'`, WHEN renderiza, THEN mostra: Nome + Gênero, Persona (4 eixos com valores), Anti-patterns (lista), Técnicas (checks), Emojis config, Voz config, Capabilities (checks).
6. GIVEN `artifact.type === 'knowledge_base'`, WHEN renderiza, THEN mostra: Lista de documentos indexados (título + chunkCount), Notas adicionais, Domain answers.
7. GIVEN `artifact.type === 'final_summary'`, WHEN renderiza, THEN mostra: Nome do agente, Função, 6-7 bullets consolidando os outros artefatos. (Este aparece APENAS na etapa Criação.)

### 3 ações (footer)

8. GIVEN artefato com `status: 'generated' | 'regenerated'`, WHEN footer, THEN mostra: `[✏️ Refinar artefato]` (outline), `[💬 Mandar alteração no chat]` (ghost), `[✓ Aprovar]` (default primary).
9. GIVEN click "Refinar artefato", WHEN callback, THEN chama `onRefine(artifactId)` (parent decide: inline expansion na 09.7 ou Dialog na 09.8).
10. GIVEN click "Mandar alteração no chat", WHEN callback, THEN chama `onChatChange(artifactId)` (parent foca composer + insere placeholder).
11. GIVEN click "Aprovar", WHEN callback, THEN chama `onApprove(artifactId)` (parent chama server action, atualiza status).

### Estados especiais

12. GIVEN `status: 'approved'`, WHEN renderiza, THEN: sem botões, check verde 16px no canto superior direito, conteúdo com `opacity-70`, hover revela 100% opacity.
13. GIVEN `status: 'regenerated'` durante stream do LLM (estado UI transient), WHEN flag `isRegenerating`, THEN substitui conteúdo por 3-5 linhas shimmer animadas + label `[regenerando]`.

### Keyboard shortcuts

14. GIVEN card focado (Tab), WHEN user aperta `R`, THEN trigga `onRefine`. `C` trigga `onChatChange`. `A` trigga `onApprove`.
15. GIVEN card approved, WHEN focado, THEN shortcuts não funcionam (aria-disabled).

### Accessibility

16. GIVEN card, WHEN screen reader, THEN `role="article"` + `aria-labelledby` apontando pro título.
17. GIVEN botões, WHEN renderiza, THEN `aria-label` descritivo (ex: "Refinar artefato Perfil do Negócio").

### Integração no chat

18. GIVEN mensagem do Arquiteto com card embutido (via message part tipo `artifact-reference`), WHEN renderiza no `ArchitectMessage`, THEN inclui texto antes do card + card + texto depois.
19. GIVEN message da AI SDK recebe parts como array, WHEN parse, THEN detecta parts custom tipo `{ type: 'artifact', artifactId }` e renderiza `ArtifactCard` no lugar.
20. GIVEN card approved avança etapa, WHEN parent detecta mudança de `currentStage` via Realtime, THEN StatusBar atualiza check verde.

### Translations

21. GIVEN strings do card, WHEN i18n, THEN `architect.artifacts.*` chaves (UI Spec § 14).
