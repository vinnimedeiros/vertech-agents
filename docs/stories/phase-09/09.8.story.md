# Story 09.8 — ArtifactDialogRefinement (Blueprint do Agente)

**Status:** Draft
**Phase:** 09
**Owner agent:** `@dev` (Neo)
**Story type:** UI component + complex form + mobile bottom sheet
**Estimated effort:** 4-5h

## Context

Blueprint do Agente tem 12+ campos (persona 4 eixos, anti-patterns, 6 técnicas, emoji config, voice config, 5 capabilities). Inline expansion ficaria claustrofóbico. Padrão híbrido C aprovado: usa Dialog centralizado (desktop) ou Bottom Sheet (mobile).

Esta story entrega o Dialog + Bottom Sheet + form completo com 7 seções colapsáveis.

**Specs de referência:**
- UI Spec § 6.3 (Dialog wireframe): `docs/phase-09/ui-spec-arquiteto.md`
- UI Spec § 6.4 (mobile bottom sheet)
- UI Spec § 5.5.2 (conteúdo completo do Blueprint)

## Acceptance Criteria

### Trigger + layout desktop

1. GIVEN `ArtifactCard` tipo `agent_blueprint`, WHEN user clica "Refinar", THEN parent decide pattern Dialog (híbrido).
2. GIVEN Dialog `shadcn/ui`, WHEN abre, THEN `max-w-[720px] max-h-[85vh]` com scroll interno.
3. GIVEN Dialog, WHEN overlay, THEN fundo `bg-background/80` escurece chat atrás.
4. GIVEN Dialog, WHEN user aperta ESC, THEN fecha = Cancelar.
5. GIVEN Dialog, WHEN user clica fora, THEN fecha = Cancelar.

### 7 seções colapsáveis

6. GIVEN form, WHEN renderiza, THEN 7 seções (`Accordion` shadcn), todas expandidas por default na primeira abertura:
   - **Identidade**: Nome (Input) + Gênero (RadioGroup Feminino/Masculino)
   - **Personalidade**: 4 Sliders 0-100 com labels (Tom, Formalidade, Humor, Empatia). Reuso `LabeledSlider` de 07B.4 v1.
   - **Anti-patterns**: TagList com reuso 07B
   - **Técnicas comerciais**: Checkbox list (6 presets: Rapport, SPIN, AIDA, PAS, Objeção, Follow-up) + select per-item de intensidade (soft/balanced/aggressive)
   - **Emojis**: RadioGroup modo (nenhum/curadoria/livre) + quando curadoria: lista de emojis escolhidos + checkboxes de usage rules (allowed in: saudação/celebração/conquista/empatia; forbidden in: preço/objeção/reclamação/tópico sério)
   - **Voz**: Toggle enabled; se enabled: Select provider (ElevenLabs/Qwen) + Input voice ID + RadioGroup mode + (se triggered) Checkbox triggers
   - **Capabilities**: Checkbox list (Qualificação, Agendamento, FAQ, Handoff, Follow-up)

### Form state

7. GIVEN form, WHEN inicializado, THEN preenche com valores atuais do artefato via `content`.
8. GIVEN user muda campo, WHEN form dirty, THEN botão "Salvar alterações" habilitado.

### Actions (footer sticky)

9. GIVEN Dialog, WHEN footer, THEN `[Cancelar]` (ghost esquerda) + `[Salvar alterações]` (primary direita), fixos.
10. GIVEN save válido, WHEN submit, THEN chama `updateArtifactInline` (reuso 09.7) com content novo.

### Validation (zod)

11. GIVEN schema zod, WHEN valida, THEN sliders 0-100 int, gênero enum, arrays com limite (anti-patterns max 20), voice enabled requer voiceId quando triggered.
12. GIVEN validation errors, WHEN submit, THEN errors inline nos campos, scroll até primeira seção com erro.

### Mobile (bottom sheet)

13. GIVEN breakpoint < 768px, WHEN renderiza, THEN usa `Sheet side="bottom"` com `max-h-[90vh]` + drag handle visível.
14. GIVEN mobile bottom sheet, WHEN drag down, THEN fecha = Cancelar.
15. GIVEN mobile footer, WHEN sticky bottom, THEN [Cancelar] + [Salvar] mantém mesma largura.

### Accessibility

16. GIVEN Dialog, WHEN screen reader, THEN `role="dialog"` com `aria-labelledby` no título.
17. GIVEN form, WHEN Tab navega, THEN ordem lógica: seção por seção, top-down.
18. GIVEN ESC fecha, WHEN user aperta, THEN toast confirma "Alterações descartadas".

### Reuso

19. GIVEN componentes existentes, WHEN monta form, THEN reusa: `LabeledSlider` (07B.4), `TagList` (07B), `RadioGroup`, `Checkbox`, `Input`, `Select`, `Switch`, `Dialog`, `Sheet`, `Accordion` (todos shadcn).

### Translations

20. GIVEN strings, WHEN i18n, THEN `architect.artifacts.refinementDialog.*` + `architect.blueprintForm.*`.
