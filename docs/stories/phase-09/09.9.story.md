# Story 09.9 — FlowDiagramPreview + publishAgentFromSession

**Status:** Draft
**Phase:** 09
**Owner agent:** `@dev` (Neo)
**Story type:** React Flow + backend transaction trigger
**Estimated effort:** 4-5h

## Context

Finaliza o fluxo de criação. Na etapa Criação final, depois do Resumo Final aprovado, aparece preview visual read-only do Flow Diagram do agente que será criado (Orquestrador → Agente → Categorias → Tools). User clica "Criar agente" e dispara a transação atômica (tool `publishAgentFromSession` da 08A.3), cria agent em DRAFT, redireciona pro Painel de Refino.

**Specs de referência:**
- UI Spec § 8 (Flow Diagram preview wireframe): `docs/phase-09/ui-spec-arquiteto.md`
- Tech Spec § 6 (transação atômica completa): `docs/phase-09/tech-spec-arquiteto.md`
- Research § 5 (React Flow @xyflow/react v12 + Dagre): `docs/phase-09/research-dependencies.md`

## Acceptance Criteria

### Setup React Flow

1. GIVEN dependencies, WHEN `pnpm add @xyflow/react @dagrejs/dagre`, THEN instalação OK (Atlas confirmou em research).
2. GIVEN import, WHEN `import { ReactFlow, ReactFlowProvider, Background } from '@xyflow/react'`, THEN compila.
3. GIVEN CSS, WHEN `import '@xyflow/react/dist/style.css'`, THEN estilos aplicados.

### FlowDiagramPreview component

4. GIVEN `FlowDiagramPreview` read-only, WHEN config, THEN `nodesDraggable={false} nodesConnectable={false} elementsSelectable={false} panOnDrag={false} zoomOnScroll={false} zoomOnPinch={false} zoomOnDoubleClick={false} preventScrolling={false} fitView`.
5. GIVEN estrutura, WHEN renderiza nodes, THEN: 1 Orquestrador (dim, opacity-40, placeholder), 1 Agente (card central com avatar+nome), 5 Categorias (Qualificação/Agendamento/FAQ/Handoff/Follow-up se ativo), N Tools (folhas baseadas em `agent.enabledTools`).
6. GIVEN edges, WHEN renderiza, THEN Orquestrador → Agente → cada Categoria → Tools, fino `stroke-muted-foreground/30`.
7. GIVEN layout Dagre, WHEN aplicado, THEN direction TB (top-down), ranksep 80, nodesep 40.

### Animações de entrada

8. GIVEN nodes entram, WHEN render, THEN `fade-in + slide-up` com stagger 100ms entre cada node (CSS keyframes).
9. GIVEN edges, WHEN animam, THEN dasharray animated 20s linear infinite.
10. GIVEN `prefers-reduced-motion`, WHEN user opta por reduce, THEN desativa animações via CSS media query.

### Container visual

11. GIVEN container, WHEN renderiza, THEN `bg-muted/20 border rounded-lg p-6 max-w-[640px]`.
12. GIVEN background, WHEN ReactFlow, THEN `<Background color="#aaa" gap={16} />`.
13. GIVEN attribution, WHEN licença, THEN `proOptions={{ hideAttribution: true }}` (OSS aceito).

### Client vs Server

14. GIVEN `'use client'` no topo, WHEN component, THEN React Flow só renderiza client-side.
15. GIVEN server component parent, WHEN renderiza, THEN usa `<Suspense fallback={<Skeleton />}>` pra loading state.

### Placement dentro da mensagem do Arquiteto

16. GIVEN etapa Criação, WHEN `final_summary` aprovado, THEN Arquiteto envia mensagem com: texto contextual + `<ArtifactCard type="final_summary">` + `<FlowDiagramPreview>` + `<CreateAgentCTA>`.
17. GIVEN `agent_creation_session.status !== 'DRAFT'`, WHEN já publicou, THEN esconde CTA, mostra "Agente criado".

### Create Agent CTA

18. GIVEN botão `[✨ Criar agente {name}]`, WHEN renderiza, THEN `Button size="lg" variant="default"` centralizado com ícone Sparkles.
19. GIVEN click, WHEN handler, THEN chama server action `publishAgentAction(sessionId)`.
20. GIVEN state, WHEN loading, THEN button vira "Criando..." com spinner.

### Server action `publishAgentAction`

21. GIVEN server action em `apps/web/modules/saas/agents/architect/lib/actions.ts`, WHEN invocada, THEN chama tool `publishAgentFromSession(sessionId)` (08A.3) OU replica a lógica de transação atômica direto no server action (tech spec § 6).
22. GIVEN sucesso, WHEN retorna, THEN `{ success: true, agent }`, redireciona cliente pra `/agents/[agentId]` com toast `"Agente {name} criado com sucesso"`.
23. GIVEN falha de validação (checklist incompleto), WHEN retorna, THEN toast destructive + Arquiteto envia mensagem explicando o que falta.
24. GIVEN falha de transação, WHEN rollback, THEN mensagem no chat + botão "Tentar novamente".

### Retry strategy

25. GIVEN falha transient, WHEN 1º retry, THEN botão CTA muda pra "Tentar novamente" + toast info.
26. GIVEN 3 tentativas falhas, WHEN persistente, THEN mostra código erro `ERR-{sessionId}` no chat.

### Performance

27. GIVEN Flow Diagram com ~10-20 nodes, WHEN renderiza, THEN nenhum jank perceptível (sob 16ms por frame).
28. GIVEN nodeTypes/edgeTypes, WHEN definidos, THEN constante module-level (evita re-render).

### Translations

29. GIVEN strings, WHEN i18n, THEN `architect.creation.*` (UI Spec § 14).
